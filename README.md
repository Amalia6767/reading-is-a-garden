# Reading is a Garden
#### 阅读是一座花园

> 每一个工具都在帮你读懂眼前这一篇，却没有一个，让你明天还愿意翻开下一篇。
> 这座花园，为的是后一个问题。
>
> *Every tool makes one paper easier to understand. None of them make you want to come back tomorrow. This garden is built for the second problem.*

---

## 缘起 · Origin

读论文一直是件难事——不是难在读不懂，而是难在**读不下去**。一篇 AI 或化学的论文，常常翻了一两页就搁下了，第二天不会再打开。后来我发现，让人弃坑的从来不是某一篇的难度，而是"明天还要不要再来"这件事本身。

市面上的阅读工具，几乎都在优化同一件事：**让你读懂这一篇**——摘要、翻译、问答、逐段精读。可没有一个在意**留存**：读完这一篇之后，你还愿不愿意读下一篇。这正是 Duolingo 的洞察——它赢的地方从不是把语言教得多好，而是让你连续三百天回来。

于是有了这座花园。

你读过的每一篇论文，都会在这里长成一株会发光的植物。读得越多，花园越繁盛；连续地读，它便持续生长、直至开花。**进度不该是一根冷冰冰的进度条，而该是一片会生长、值得驻足、甚至美到想与人分享的风景。** 把"读论文"这件苦差事，变成一件看得见的作品——这是这个项目真正的野心，也是它想守住的东西。

## 它如何运作 · How it works

```
一篇论文
   │
   ├─ 切成一瓣一瓣，每次读一瓣        备料：不把整篇甩给你，Duolingo 从不让你一天学完一门语言
   │
   ├─ 园丁零基础讲解                  为什么 → 是什么 → 最后才碰公式，每一步都配直觉与类比
   │
   ├─ 沉淀成花园里的一株植物          你的努力不蒸发，化作一份可回访的笔记
   │
   └─ 连续天数 · 生长 · 开花          给你明天回来的理由
```

**生长的唯一养分是阅读本身。** 时间不会替你浇灌，鼠标也不能。一株花只有在你把整篇论文读完的那一刻，才获得完全绽放的资格——所以，**花开，意味着你真的读完了一篇**。

## 花园 · The Garden

花园是这个项目的美学核心，也是它的护城河。它是一个自包含、可离线打开的网页：

- **一场序幕。** 初次推开花园，是一粒种子的旅程：一个词、一点光，坠入土中、在黑暗里扎根、破土、向上，最终绽放，散作满原星火——落进你真实的花园。一镜到底，通篇只有夜的银蓝，与那粒种子唯一的一点暖光。
- **永远的夜。** 花园长在一场不散的月夜里——会发光的花，只属于黑暗。月亮仍随你这一天缓缓西移，时间没有停，只是永远停在夜里。你读过的植物在暗中泛起生物荧光；未读的，只是静默的剪影。
- **一株一世界。** 每株植物的形态与配色，由那篇论文的标题生成，独一无二。花有三种性情：雏菊、百合、与一团光丝编成的光球。
- **提灯与凝视。** 光标是一盏提灯，照到哪里，哪里的植物便苏醒、舒展、发亮。长按，是为它注入光、亲手将它养高。移动视线，五层景深随之错动，花园便有了呼吸与纵深。
- **有生灵栖居。** 萤火追着你的提灯，一路明灭；飞蛾绕着月亮打转。夜是安静的，却不空。
- **每一朵都是一份档案。** 点开一株植物，它舒展成一页"知识标本"——一侧是它的植物学图鉴（拟真的学名、花冠、生长档案），一侧是园丁为你写下的完整讲解。知识与花，在这里合为一体。

## 快速开始 · Quick Start

```bash
# 1 · 环境
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt

# 2 · 密钥（在 https://platform.deepseek.com 申请）
cp .env.example .env            # 编辑 .env，填入你的 DeepSeek key

# 3 · 播种：把论文（PDF / TXT / Markdown）放进 seeds/ 文件夹

# 4 · 读今晚的一瓣
.venv/bin/python agent.py       # 无需指定读哪篇——园丁替你决定，回车即读

# 5 · 逛你的花园
.venv/bin/python garden_web.py open    # 加 demo 可看满园繁花 + 开场序章
```

园丁会自动挑选：先接着读没读完的那篇，读完了再开一颗新种子。你每天只需重复第 4 步。

## 路线图 · Roadmap

这是一个**公开生长**的项目：我在一边学工程、一边把它造出来，过程本身也是记录的一部分。

- **v0.1** — 读 → 讲解 → 沉淀为植物 → 连续打卡 ·  已完成
- **v0.2** — 昼夜生物荧光花园，可交互、会生长的可视化 ·  已完成
- **v0.3** — 种子箱、把论文切分成瓣、以阅读驱动生长的经济系统 ·  已完成
- **v0.4** — 电影镜头：点开一朵花，镜头俯冲扎进花心 ·  已完成
- **v0.5** — 开场序章「一粒种子的旅程」，花园入永夜 ·  已完成
- **接下来** — 费曼式自测（读完考三问，答不出即暴露没真懂）
- **再往后** — 知识图谱式布局：主题相近的论文，花也开得相近，让"读过的东西如何彼此关联"被看见

> 注：v0.2 曾做过随本地时间流转的完整昼夜，v0.5 起花园定格为永夜——会发光的花只属于黑暗。
